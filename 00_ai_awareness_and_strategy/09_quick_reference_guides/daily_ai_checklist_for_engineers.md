# Daily AI Checklist for Engineers

**Quick Reference Guide | One-Page Cheat Sheet**

**Purpose:** Daily checklist for engineers using AI tools

---

## Morning Routine

### Tool Setup
- [ ] GitHub Copilot/Cursor active and configured
- [ ] ChatGPT/Claude accessible for Q&A
- [ ] AI tools authenticated and ready

### Review
- [ ] Check for AI-generated code reviews from yesterday
- [ ] Review any AI suggestions or alerts
- [ ] Verify AI tool updates/patches

---

## During Development

### Code Generation
- [ ] Use Copilot/Cursor for code completion
- [ ] Review all AI-generated code before accepting
- [ ] Verify AI suggestions match requirements
- [ ] Test AI-generated code thoroughly

### Code Review
- [ ] Review AI-generated code for security issues
- [ ] Check for business logic correctness
- [ ] Verify error handling is appropriate
- [ ] Ensure code follows team standards

### Testing
- [ ] Use AI to generate test cases when appropriate
- [ ] Review AI-generated tests for coverage
- [ ] Verify test cases are meaningful and correct
- [ ] Run AI-generated tests before committing

---

## Security & Compliance

### Data Security
- [ ] No sensitive data (SSNs, passwords) in AI tools
- [ ] Use placeholders (`<EXAMPLE_ONLY>`) for sensitive data
- [ ] Verify AI outputs don't expose sensitive information
- [ ] Check AI tool data handling policies

### Code Security
- [ ] Review AI-generated code for security vulnerabilities
- [ ] Check for SQL injection, XSS, authentication issues
- [ ] Verify API keys and credentials are not hardcoded
- [ ] Ensure proper error handling (no information leakage)

---

## Quality Assurance

### Verification
- [ ] All AI-generated code tested
- [ ] Business logic verified (especially for payroll/tax)
- [ ] Edge cases handled correctly
- [ ] Error messages are appropriate

### Documentation
- [ ] AI-generated documentation reviewed
- [ ] Code comments are accurate
- [ ] API documentation is complete
- [ ] Examples are correct

---

## End of Day

### Review
- [ ] Review AI tool usage for the day
- [ ] Note any issues or improvements needed
- [ ] Update team on AI tool effectiveness
- [ ] Plan AI usage for tomorrow

### Learning
- [ ] Note effective prompts that worked well
- [ ] Document AI tool tips and tricks
- [ ] Share learnings with team
- [ ] Update personal AI best practices

## Weekly Tasks

- [ ] Review AI tool performance and productivity gains
- [ ] Update AI tool configurations if needed
- [ ] Share AI tool learnings with team
- [ ] Review and update AI best practices
- [ ] Check for AI tool updates and new features
