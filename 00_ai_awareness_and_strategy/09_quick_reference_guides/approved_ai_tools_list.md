## Data Security Rules

⚠️ **Never Paste:**
- Employee SSNs, salaries, personal information
- Customer financial data
- Passwords, API keys, credentials
- Proprietary business information

✅ **Use Placeholders:**
- `<EXAMPLE_ONLY>` for sensitive data
- `<EMPLOYEE_ID>` for employee identifiers
- `<SALARY_AMOUNT>` for salary data

---

## Approval Process

1. Submit tool request to IT/Security
2. Security review (data handling, compliance)
3. Pilot testing (limited scope)
4. Approval/rejection decision
5. Documentation and training

**Contact:** IT/Security team for tool requests

---

## Tool Usage Guidelines

- ✅ Use approved tools for their intended purpose
- ✅ Follow data security rules (no sensitive data)
- ✅ Review AI outputs before using in production
- ✅ Report security concerns immediately
- ❌ Don't use unapproved tools
- ❌ Don't paste sensitive data
- ❌ Don't bypass security controls
