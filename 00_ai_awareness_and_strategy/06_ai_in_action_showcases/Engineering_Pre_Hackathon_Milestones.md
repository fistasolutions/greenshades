# Engineering Department: Pre-Hackathon Milestones

**Department:** Engineering  
**Hackathon:** AI Code Generation & Spec-Driven Development  
**Duration:** 8 Weeks  
**Target Audience:** Software Engineers, Senior Developers, Engineering Managers

---

## Overview

This 8-week pre-hackathon program prepares Engineering teams for the AI Agentic AI Hackathon by building foundational knowledge, tool proficiency, and practical skills in AI-assisted development, spec-driven development, and MCP.

---

## Milestone 1: AI Fundamentals (Weeks 1-2)

### Learning Objectives
- Understand AI and agentic AI concepts
- Learn how AI assists in software development
- Understand MCP and its role in AI agents
- Complete foundational training

### Required Curriculum Modules
- ✅ `00_introduction_to_ai_and_agentic_ai/` (all 6 lessons)
- ✅ `01_ai_vs_ai_assisted_vs_ai_driven/` (all 4 lessons)
- ✅ `03_mcp_and_specdriven_development/00_what_is_mcp.md`
- ✅ `03_mcp_and_specdriven_development/01_why_mcp_for_ai_agents.md`

### Hands-On Exercises

**Exercise 1: AI Tool Setup**
- [ ] Install and configure GitHub Copilot
- [ ] Install and configure Cursor
- [ ] Set up Claude Code account
- [ ] Test basic code generation with each tool

**Exercise 2: First AI-Generated Code**
- [ ] Generate a simple function using Copilot
- [ ] Generate a React component using Cursor
- [ ] Generate a Python class using Claude Code
- [ ] Compare outputs from all three tools

**Exercise 3: Understanding Agentic AI**
- [ ] Read MCP documentation
- [ ] Understand agent orchestration concepts
- [ ] Review case study: Payroll Analyzer Agent
- [ ] Identify potential agent use cases in your work

### Assessment Checkpoint
- [ ] Complete all curriculum modules
- [ ] Successfully generate code with all three tools
- [ ] Can explain difference between AI-assisted and agentic AI
- [ ] Understand MCP basics

**Readiness Criteria:** ✅ All exercises completed, all assessments passed

## Milestone 2: Tool Proficiency (Weeks 3-4)

### Learning Objectives

### Required Curriculum Modules
- ✅ `05_code_generation/readme.md` (complete guide)

- ✅ `01_prompt_engineering_for_everyone/readme.md` (prompt engineering)

### Hands-On Exercises

### Assessment Checkpoint

**Readiness Criteria:** ✅ All exercises completed, tool proficiency achieved

## Milestone 4: Hackathon Readiness (Weeks 7-8)

### Learning Objectives
- Review hackathon problem statements
- Form teams and assign roles
- Prepare development environment
- Final skill validation

### Required Activities

**Week 7: Problem Statement Review**
- [ ] Review all 5 Engineering hackathon problem statements
- [ ] Research each problem domain
- [ ] Identify preferred problem to solve
- [ ] Prepare initial approach/design

**Week 7: Team Formation**
- [ ] Form teams (3-5 people per team)
- [ ] Assign roles (lead developer, tester, documenter)
- [ ] Set team communication channels
- [ ] Schedule team meetings

**Week 8: Environment Preparation**
- [ ] Set up development environment
- [ ] Configure all AI tools
- [ ] Prepare code repository
- [ ] Test tool integrations

**Week 8: Final Validation**
- [ ] Complete mock hackathon exercise
- [ ] Generate code for sample problem
- [ ] Present to team for feedback
- [ ] Refine approach based on feedback

### Assessment Checkpoint
- [ ] Team formed and roles assigned
- [ ] Development environment ready
- [ ] Problem statement selected
- [ ] Initial design/approach prepared
- [ ] Mock exercise completed successfully

**Readiness Criteria:** ✅ All preparations complete, ready for hackathon
